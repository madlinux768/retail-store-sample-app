Scenario D: Security Group Block

Network Engineer Connectivity down but TGW looks healthy
Cross-VPC connectivity is down again but when I look at the transit gateway route table, the routes look correct — no blackholes. Packets seem to be reaching the partner VPC. Can you check the VPC flow logs in the partner VPC for any REJECT entries? Maybe something changed in the security groups.
